The fastest way to try Rolter is the published Docker image — it starts the gateway, dashboard, and built-in `fake-llm` without a provider account. If you prefer a binary install, see the [Installation guide](/installation). By the end of this guide you'll have a running gateway and a verified response from your first AI request.

<Steps>
  <Step title="Install Rolter">
    Make sure you have [Docker](https://docs.docker.com/get-docker/) installed, then pull the image.

    ```bash
    docker pull ghcr.io/rolter-ai/rolter:latest
    ```

    <Tip>
      Prefer a local binary? The [Installation page](/installation) covers `cargo install` and `uv tool install rolter` as alternatives.
    </Tip>
  </Step>
  <Step title="Start Rolter">
    Start the zero-config local stack.

    ```bash
    docker run --rm -p 4000:4000 -p 4001:4001 ghcr.io/rolter-ai/rolter:latest
    ```

    The default single-container command is `rolter easy-up`: gateway on port 4000 and the dashboard on port 4001. It also includes the deterministic `fake-llm` route, so no provider credential or config file is needed for the first request.

  </Step>
  <Step title="Use a real provider (optional)">
    Mount an annotated config file and provide a provider key when you are ready to route real traffic:

    ```bash
    cp rolter.example.toml rolter.toml
    docker run --rm -p 4000:4000 -p 4001:4001 \
      -e OPENAI_API_KEY \
      -v "$PWD/rolter.toml:/app/rolter.toml" \
      ghcr.io/rolter-ai/rolter:latest
    ```

    Once the containers are running:

    - **Gateway** (data plane): [http://localhost:4000](http://localhost:4000)
    - **Control plane \+ dashboard**: [http://localhost:4001](http://localhost:4001)

    <Tip>
      Need Postgres, Redis, ClickHouse, and separate gateway/control processes? Use the full Compose deployment: `docker compose -f docker/docker-compose.yml up -d`.
    </Tip>
  </Step>
  <Step title="Send a chat completions request">
    With the gateway up, call the built-in `fake-llm` model. It needs no API key and is intended for smoke tests and local development.

    <CodeGroup>

    ```bash cURL
    curl -s http://localhost:4000/v1/chat/completions \
      -H "Content-Type: application/json" \
      -d '{"model":"fake-llm","messages":[{"role":"user","content":"hello"}]}'
    ```

    ```python Python (OpenAI SDK)
    from openai import OpenAI
    
    client = OpenAI(
        base_url="http://localhost:4000/v1",
        api_key="unused",
    )
    
    response = client.chat.completions.create(
        model="fake-llm",
        messages=[{"role": "user", "content": "Hello!"}],
    )
    print(response.choices[0].message.content)
    ```

    ```bash cURL (Anthropic)
    curl -s http://localhost:4000/v1/messages \
      -H "Content-Type: application/json" \
      -d '{"model":"fake-llm","max_tokens":256,"messages":[{"role":"user","content":"hello"}]}'
    ```

    </CodeGroup>

    You should receive a standard OpenAI or Anthropic response object — exactly what your existing code already expects.

    <Info>
      Streaming works too. Add `"stream": true` to your request body and Rolter will forward the SSE stream from the upstream provider directly to your client.
    </Info>
  </Step>
  <Step title="Verify with the models endpoint">
    Confirm the gateway is healthy and listing its built-in route:

    ```bash
    curl -s http://localhost:4000/v1/models
    ```

    The response includes `fake-llm`. When you mount `rolter.toml`, it will also list the public model names you configure.

    You can also hit the liveness endpoint at any time without a key:

    ```bash
    curl -s http://localhost:4000/healthz
    ```

    <Note>
      A Prometheus-format metrics endpoint is available at `GET /metrics` for integration with Grafana, Datadog, or any other scraper.
    </Note>
  </Step>
</Steps>
